# COURSE: Kiến trúc phần mềm (CSC13106)

## GENERAL INFORMATION
- **Course ID**: CSC13106
- **English Name**: Software Architecture
- **Vietnamese Name**: Kiến trúc phần mềm
- **Responsible Person**: for the module: Trần Văn Quý
- **Semester**: (s) in which the module is taught: 7
- **Workload**: (incl. contact hours, self-study hours): Hours for lecture: ~ 37.5 hours Hours for laboratory session: ~ 25 hours Hours for self-study, examination preparation, other activities: ~ 116.25 hours

## COURSE DESCRIPTION
• COURSE TOPICS:
- Overview of software architecture
- Object-oriented design pattern (Creational Pattern group)
- Behavioral Patterns (group of Behavioral Patterns)
- Object-oriented design pattern (group of Structural Patterns)
- Middleware:  distributed object, methods for communicating with messages, application tier, message mapping and association, defining and implementing composite components
- Service-Oriented Architecture
- Model-Driven Architecture
- Cloud Deployment Architecture

- [3]Fundamentals of Software Architecture: An Engineering Approach,Neal Ford, Mark Richards,2020,O'Reilly Media
- [4]Microservices for the Enterprise: Designing, Developing, and Deploying,Kasun Indrasiri, Prabath Siriwardena,2018,APRESS

## LEARNING OBJECTIVES (CLOs)
- G1.Practice actively participating in group discussions (in large groups of subjects), dividing work and coordinating work according to plans in small groups (including 2-3 students);
Identify roles and responsibilities of team members
- G2.Summarize specialized documents and lectures/slides/textbooks in English in analyzing and designing software architecture; explain some English terms in software architecture analysis and design; conduct presentation (in the form of a report written according to the prescribed form) and a presentation on the group's topic.
- G3.Practice critical thinking and holistic way of thinking when analyzing and designing software architecture
- G4.Describe the meaning of software architecture, general principles in architecture analysis and design, criteria and requirements when designing and evaluating software architecture
- G5.Conduct a presententation and analyze a number of solutions on technology and software architecture; analyze the advantages and limitations of a technology solution and software architecture
- G6.Apply some software architectures introduced in the module to software development
- G7.Propose improved solutions on available technology and software architecture for software systems
- G8.Practice personal skills such as self-study, analysis and evaluation of new technology and architecture solutions

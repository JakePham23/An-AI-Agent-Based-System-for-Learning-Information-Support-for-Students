- **English Name**: Software Modeling

- **Responsible Person**: for the module: Bùi Tấn Lộc
- **Semester**: (s) in which the module is taught: 7
- **Workload**: (incl. contact hours, self-study hours): Hours for lecture: ~ 37.5 hours Hours for laboratory session: ~ 25 hours Hours for self-study, examination preparation, other activities: ~ 116.25 hours

## COURSE DESCRIPTION
As software modeling plays not only an important role in the actual coding effort, we will also practice its effects in requirements engineering and testability.
Most of the course will be hands-on. Expect to write, design, discuss, and code a lot. We could apply any agile approach such as apply Scrum for our team management and use Kanban to manage our projects. We will manage our code via a git-based configuration management tool.
• COURSE TOPICS:
- An introduction to modeling and software modeling
- Business process modeling
- User requirement modeling
- Analysis modeling
- Design modeling
- Software testing specifications
- An introduction to model driven approach and model driven engineering
- Generic textual syntax modeling
- Graphical editors modeling
- Model validation and transformation

## LEARNING OBJECTIVES (CLOs)
- G1.Explain necessary models used for different software development methodologies (approaches, processes, modeling languages and tools)
- G2.Apply personal skills and teamwork skills for making artifacts which are produced and used during a software development project.
- G3.Recognize how information technology enables to change business process to yield business value.
- G4.Apply, analyze, synthesize and evaluate how parts of the software development process are broken and modeled in their relations which help us facilitate agile or enterprise software development.
- G5.Synthesize test case and test plan specifications
